A verified description of how audio flows through the US-16x08's built-in DSP
mixer, and how that maps onto the controls this project exposes. It is the single
source behind the CLI's `topology` output, the control catalog's doc comments, and
the future GUI.

Each claim is tagged _(manual)_, _(driver)_, or _(inferred)_ — see <<sec-sources>>.

[[sec-overview]]
== Overview

The US-16x08 is a 16-in / 8-out USB interface whose on-board BlackFin DSP runs a
*monitor mixer*: the 16 inputs are processed per channel and summed into a single
*stereo master bus*. The computer's eight USB playback streams (`COMPUTER 1–8`)
can be folded into that bus and/or sent straight to outputs. Each of the eight
physical line outputs independently selects one source. _(manual)_

[[sec-channel-chain]]
== Per-input-channel chain

The per-channel stages, in signal order, one table per stage. The stages and their
roles are _(manual)_; the exact EQ-vs-compressor ordering is _(inferred)_ — see the
note. Raw control ranges are _(driver)_; the human units (dB, Hz, Q, ms) are
_(manual)_. The *Range* column shows the raw control's `min–max (default)`.

NOTE: The manual presents EQ as channel module 1 and the compressor as module 2,
and the compressor acts on the channel "input volume" — consistent with *phase →
LCF/EQ → compressor → mute → fader → pan*. The _Block diagrams_ (p. 31) depict the
analog I/O path (mic/line → preamp → A/D), not the internal DSP order, so
EQ-before-compressor is strongly implied but not drawn as an explicit DSP block.
_(inferred)_

[[sec-stage-lcf]]
=== Low-cut filter (LCF)

The device has a per-channel low-cut (high-pass) filter, but the Linux driver
registers *no control* for it, so this project cannot set it — there is no row to
list. _(driver)_

[[sec-master]]
== Master bus

The 16 channels are summed into a stereo master; the master meters show the mixed
signal. The compressor can also operate on the master stereo signal. _(manual)_

[#tbl-master]
.Master controls
[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `master-volume` | Master Volume | 0–133 (127) | Master output level (same dB curve as the channel faders).
| `master-mute` | Master Mute Switch | on / off | Mute the master output.
|===

[[sec-routing]]
== Output routing

Each `LINE OUT 1–8` selects *one* source through the `route` control (one instance
per output, indices 0–7). _(manual)_

[#tbl-route]
.Routing control
[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `route` | Line Out Route | enum 0–9, per output (0–7) | Source for one physical line output.
|===

The 10 sources fall into two groups:

Master L / Master R:: The *Stereo BUS* — "the signals input through each input jack
and by USB output from the computer are mixed and output in stereo."
COMPUTER 1–8 (this project's `Output 1..8`):: The *Computer BUS* — DAW playback
streams sent directly to a jack.

`LINE OUT 1` and `LINE OUT 2` are mirrored to the PHONES jack. _(manual)_

IMPORTANT: The 16 inputs are *not* routed to outputs individually — they are summed
into the stereo master. Only the eight outputs are routed.

[[sec-not-exposed]]
== Not exposed / out of scope

Low-cut filter (LCF):: On the device, but the kernel driver registers no control
for it. _(driver)_
Stereo LINK:: Combining two adjacent channels into a stereo pair is a
mixer-software grouping (it ganged the GTK widgets); the driver has no link
control, so on the hardware each channel is independent. This is GUI state for the
future GUI, not a hardware control. _(driver/manual)_
Mic preamp gain, phantom power, sample rate:: Front-panel / interface functions,
not DSP-mixer controls; outside this project's control surface.
