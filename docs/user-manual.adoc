This manual describes how to control the Tascam US-16x08's built-in DSP mixer
from Linux, using the two tools this project provides: the command-line tool
`rackctl-us16x08` and the graphical mixer `rackctl-us16x08-gui`.

[[sec-intro]]
== Introduction

The US-16x08 is a 16-in / 8-out USB audio interface with an on-board DSP mixer:
a 4-band equalizer, a low-cut filter, and a compressor on each input channel,
summed into a stereo master bus and routed to eight physical outputs. On Linux
the `snd-usb-audio` driver exposes that mixer as roughly 280 ALSA control
elements. The two tools in this project drive those controls.

`rackctl-us16x08`:: A command-line tool for reading and writing individual controls,
watching changes, reading the level meters, and saving or restoring whole-mixer
or single-channel presets. Suited to scripting and to quick one-off changes.

`rackctl-us16x08-gui`:: A graphical mixer with a live meter bridge, a focused
channel editor showing the EQ and compressor as response curves, an output
panel, and an output-routing tab. Suited to interactive mixing.

Both tools act on the same hardware controls and share the same preset and
default-preset files, so they can be used interchangeably.

This manual applies to the Tascam US-16x08 mixer tools version
{applies-to-version}. It is for
anyone operating the interface on Linux. Read <<sec-glance>> first for the mental
model, then the chapter for the tool you use (<<sec-cli>> or <<sec-gui>>). The
reference chapters (<<sec-cli-ref>>, <<sec-gui-ref>>) and the verified control
catalog (<<sec-appendix-catalog>>) are for looking up exact details.

[[sec-scope]]
== Scope

This manual covers operating the US-16x08 DSP mixer through `rackctl-us16x08` and
`rackctl-us16x08-gui` on Linux.

In scope:

* Reading and writing the mixer controls (faders, pan, phase, mute, EQ,
  compressor, routing, master, global switches).
* Reading the level and gain-reduction meters.
* Saving, restoring, and recalling presets, including the shared default preset.

Out of scope:

* *Audio streaming and recording.* The tools control the DSP mixer only; they do
  not capture or play back audio. Recording and playback are handled by the
  Linux audio stack (the `snd-usb-audio` driver and your DAW). See
  <<sec-capture>> for where the recorded signal is taken from.
* *Front-panel and interface functions* not exposed as DSP controls: mic preamp
  gain, phantom power, and sample rate. These are set on the unit or by the audio
  stack, not by these tools.
* *The `rackctl-us16x08` library API.* Building applications against the control
  library is a developer topic; see the project README.

Prerequisites:

* A Linux system running the `snd-usb-audio` driver (kernel 4.11 or later, which
  added US-16x08 support).
* A connected US-16x08, recognized as ALSA card id `US16x08`.
* A built `rackctl-us16x08` and/or `rackctl-us16x08-gui` binary (see <<sec-install>>).

For trying the tools without the hardware, every command and the whole GUI run
against an in-memory mock device (see <<sec-mock>>).

[[sec-glance]]
== The mixer at a glance

This chapter builds the mental model the rest of the manual relies on: how audio
flows through the device, what the computer records, how controls are named and
valued, and what presets and stereo link mean.

[[sec-signal-flow]]
=== Signal flow

Each of the 16 inputs passes through a per-channel chain (phase, low-cut filter,
EQ, compressor, mute, fader, pan) and is summed into a stereo master bus. The
computer's eight USB playback streams (`COMPUTER 1`...`COMPUTER 8`) can be folded
into that bus or sent straight to outputs. Each of the eight physical line
outputs independently selects one source.

.US-16x08 signal flow
[plantuml, fig-signal-flow, svg]
----
@startuml
left to right direction
skinparam monochrome true
skinparam shadowing false
skinparam rectangle {
  BackgroundColor white
}

rectangle "16 inputs"                    as IN
rectangle "phase"                        as PH
rectangle "LCF + EQ"                     as EQ
rectangle "compressor"                   as CO
rectangle "fader"                        as FA
rectangle "pan"                          as PA
rectangle "MASTER bus\nL / R"            as MB
rectangle "route\n(per output)"          as RT
rectangle "line out 1..8"                as OUT
rectangle "computer playback\n(COMPUTER 1..8)" as PB
rectangle "computer capture\n(record, 16 ch)"  as CAP

IN -> PH
PH -> EQ
EQ -> CO
CO -> FA
FA -> PA
PA -> MB
MB -> RT
RT -> OUT
PB --> MB : buss-out
PB --> RT : direct
IN --> CAP : capture (dry, pre-DSP)
@enduml
----

The EQ and compressor are a per-channel module pair: the equalizer first, then
the compressor acting on the equalized signal. The `dsp-bypass` switch skips the
EQ and compressor for dry monitoring. The 16 channels are not routed to outputs
individually; they are summed into the stereo master, and only the eight outputs
are routed. The verified per-stage detail is in <<sec-appendix-catalog>>.

[[sec-capture]]
=== What the computer records

The DSP is a monitor processor. The 16 channels the device sends to the computer
for recording are taken *before* the DSP block: the recorded signal is the dry
input, so the EQ, compressor, and fader change monitoring and the outputs but
never the recording. There is no control to record the processed signal or to
move the capture point. To capture the EQ or compressor, record the processed
monitor mix another way (for example, route the stereo master to an output and
record that) or apply the processing in the DAW.

[[sec-controls-model]]
=== Controls: scope, kind, and display units

Every control has a short *key* (for example `eq-low-volume`), used by
`rackctl-us16x08` and shown by the `list` command. Each control has a *scope* and a
*kind*.

Scope says how many instances a control has:

* *global*: one instance (for example `master-volume`).
* *channel*: one per input channel, 16 in all, selected with the channel index
  0...15.
* *output*: one per physical output, 8 in all, selected with the output index
  0...7.

Kind says what values a control takes:

* *bool*: on or off (for example `mute`).
* *int*: an integer in a fixed range (for example a fader or an EQ gain).
* *enum*: one of a fixed set of labelled values (for example `comp-ratio` or
  `route`).
* *meter*: a read-only block of level values.

Integer controls are read and written in *display units* rather than raw control
numbers: dB for the faders, EQ gains, and compressor threshold and gain; Hz or
kHz for the EQ band frequencies; ms for the compressor attack and release; and
`C` / `L..%` / `R..%` for pan. Both tools use the same units, so a value shown by
`rackctl-us16x08 get` matches the value shown on the matching GUI control. The raw
ranges behind these units are listed in <<sec-appendix-catalog>>.

[[sec-presets-concept]]
=== Presets and the default preset

A *preset* is a saved set of control values stored as a JSON file. Two kinds
exist:

* A *mixer preset* holds the whole mixer: the master controls, the output
  routing, and all 16 channel strips. Saved from the GUI, it also records the
  stereo-link grouping (an extra field `rackctl-us16x08` ignores).
* A *strip preset* holds one channel's controls and can be applied to any
  channel.

The *default preset* is a mixer preset kept in a fixed location in the user
configuration directory. Both tools save to and load from the same default-preset
file, so a default saved in one tool is recalled by the other. Use it to keep a
known starting state that one command or one button restores.

[[sec-link-concept]]
=== Stereo link

A *stereo link* groups an adjacent channel pair (1 and 2, 3 and 4, and so on)
into one stereo strip. Linking is a feature of the GUI, not a hardware control:
the device treats every channel independently. When a pair is linked, the GUI
shows one strip and writes each edit to both channels. The pair is panned hard
L/R (lower channel left, upper channel right) for a stereo image, and the pan
control becomes a *balance*: the Volume fader sets the common level of both
channels, and Balance attenuates one channel's level to favour the other side
(at centre both sit at the common level; full left silences the right channel,
full right silences the left). Balance is a level adjustment, not a pan, so the
stereo image is preserved. The link state is remembered between GUI sessions and
is saved inside each whole-mixer preset and scene, so loading one restores its
grouping.

[[sec-install]]
== Installing the tools

Both tools are Rust binaries built from the project workspace.

[[sec-build]]
=== Building from source

Prerequisites: a Rust toolchain (edition 2024, 1.95 or later) and the ALSA
development headers, which the default build links against.

. Install the ALSA headers. On Debian or Ubuntu: `sudo apt-get install
  libasound2-dev`. On Arch: `sudo pacman -S alsa-lib`.
. Build the tools from the workspace root:
+
[source,bash]
----
cargo build --release
----
+
This produces `target/release/rackctl-us16x08` and `target/release/rackctl-us16x08-gui`.

To build a single tool, add `-p rackctl-us16x08-cli` or `-p rackctl-us16x08-gui`.

[[sec-mock]]
=== Running without hardware

Both tools accept an in-memory mock device that needs no card and no ALSA
headers, useful for trying commands or the interface before connecting hardware.

* `rackctl-us16x08` takes a global `--mock` flag before the command, for example
  `rackctl-us16x08 --mock list`.
* `rackctl-us16x08-gui` takes a `--mock` argument: `rackctl-us16x08-gui --mock`.

To build without the ALSA dependency at all (mock only), build with
`--no-default-features`.

[[sec-cli]]
== Using the command-line tool

`rackctl-us16x08` runs one command per invocation. A global `--mock` flag selects the
mock device; otherwise the tool opens the connected US-16x08. The exact command
and option syntax is in <<sec-cli-ref>>.

[[sec-cli-inspect]]
=== Inspecting the controls

To see every control with its key, scope, kind, and ALSA name:

[source,bash]
----
rackctl-us16x08 list
----

The KIND column shows each integer control's range in display units, for example
`int -127 dB..=+6 dB` or `int 32 Hz..=1.6 kHz`.

To explain the signal flow and how the eight outputs are routed:

[source,bash]
----
rackctl-us16x08 topology
----

To see one control in detail (scope, range in display units, enum values, and a
short description):

[source,bash]
----
rackctl-us16x08 info comp-ratio
----

`list`, `topology`, and `info` do not touch the device, so they work without
hardware and without `--mock`.

[[sec-cli-read]]
=== Reading values and meters

Read one control's current value with `get`. For a channel or output control,
give the index with `-c`:

[source,bash]
----
rackctl-us16x08 get master-volume        # a global control, prints e.g. "+0 dB"
rackctl-us16x08 get eq-low-freq -c 0     # channel 0, prints e.g. "60 Hz"
rackctl-us16x08 get mute -c 3            # channel 3, prints "true" or "false"
----

Read the level meters once with `meters`, or continuously until interrupted with
`monitor`:

[source,bash]
----
rackctl-us16x08 meters                   # one reading, dB-scaled
rackctl-us16x08 meters --raw             # linear samples instead
rackctl-us16x08 monitor                  # repeat until Ctrl-C
----

Print control changes as they happen (for example, when someone turns a knob on
the unit or another client writes a control):

[source,bash]
----
rackctl-us16x08 watch
----

NOTE: While `rackctl-us16x08-gui` is running it polls the meters continuously. Run only
one meter reader at a time; do not run `rackctl-us16x08 monitor` against the device
while the GUI is open.

[[sec-cli-write]]
=== Changing values

Write a control with `set`. Boolean, enum, and integer controls each accept
natural forms.

Booleans accept `on`/`off`, `true`/`false`, `1`/`0`, `yes`/`no`, or `toggle` to
flip the current value:

[source,bash]
----
rackctl-us16x08 set mute on -c 3
rackctl-us16x08 set mute toggle -c 3
----

Enum controls accept either the label or its index:

[source,bash]
----
rackctl-us16x08 set route "Output 3" -c 0
rackctl-us16x08 set comp-ratio 4 -c 0
----

Integer controls accept their display units. A bare number, or a number with a
unit suffix, is an absolute value:

[source,bash]
----
rackctl-us16x08 set master-volume +3dB -c 0   # dB
rackctl-us16x08 set eq-low-freq 1.2kHz -c 0    # Hz / kHz
rackctl-us16x08 set comp-release 200ms -c 0    # ms
rackctl-us16x08 set pan L50% -c 0              # C / L..% / R..%
----

A bare `+N` or `-N` adjusts the control relative to its current value, in display
units, so `+5` on a fader adds 5 dB and `+20` on `comp-release` adds 20 ms. To
set a negative absolute value, add a unit suffix so it is not read as a relative
adjust:

[source,bash]
----
rackctl-us16x08 set master-volume +5          # relative: add 5 dB
rackctl-us16x08 set eq-low-volume -6dB -c 0   # absolute: -6 dB
----

A value beyond a control's range is clamped to the range. `set` prints nothing on
success.

[[sec-cli-presets]]
=== Saving and restoring

Save the whole mixer, a single channel's strip, or just one section (the EQ or
the compressor) of a channel, to a JSON file. The `--section` option chooses how
much of the channel to save and requires `-c`:

[source,bash]
----
rackctl-us16x08 save mix.json                  # the whole mixer
rackctl-us16x08 save strip.json -c 0           # just channel 0's strip
rackctl-us16x08 save eq.json -c 0 --section eq # just channel 0's EQ
rackctl-us16x08 save comp.json -c 0 --section comp   # just channel 0's compressor
----

Restore from a file. A mixer preset restores everything and takes no channel; a
strip or section preset is applied to the channel given with `-c`, setting only
the controls the file holds:

[source,bash]
----
rackctl-us16x08 load mix.json                  # restore the whole mixer
rackctl-us16x08 load strip.json -c 5           # apply a strip to channel 5
rackctl-us16x08 load eq.json -c 5              # apply just an EQ to channel 5
----

These are the same files the GUI's scenes and channel/EQ/compressor preset tabs
read and write, so the two tools share presets. On load, the number of controls
applied is reported on standard error. Controls the device does not expose are
skipped and listed.

[[sec-cli-default]]
=== The default preset

Recall the shared default preset, or capture the current mixer as the default:

[source,bash]
----
rackctl-us16x08 default                        # load the default into the whole mixer
rackctl-us16x08 default --save                 # save the current mixer as the default
----

Loading before a default exists is an error that names the expected file. The
file is shared with the GUI's *Save default* and *Load default* buttons (see
<<sec-gui-presets>>).

[[sec-cli-default-udev]]
=== Applying the default automatically on connect

With a `udev` rule you can have the default preset applied every time the
interface is plugged in, with nothing running -- handy because the device powers
up at its own factory defaults each time it is connected. The rule reacts to the
card appearing and hands off to a small `systemd` service that runs `rackctl-us16x08
default`. (Handing off to `systemd` matters: `udev` runs commands in a stripped
environment and kills anything still running when the event finishes, so it is
the wrong place to run the tool directly.)

This complements the GUI's own behaviour: the running GUI already restores its
on-screen mix on replug (see <<sec-appendix-trouble>>); this rule covers the case
where no GUI is running.

First make sure a default exists and note where `rackctl-us16x08` is installed:

[source,bash]
----
rackctl-us16x08 default --save                 # capture the current mixer as the default
command -v rackctl-us16x08                      # e.g. /usr/local/bin/rackctl-us16x08
----

Create the `udev` rule `/etc/udev/rules.d/99-rackctl-us16x08.rules`. It matches the
card by its ALSA id (`US16x08`) and triggers the service when the control device
appears, which is the point at which the card is ready to be opened:

[source]
----
ACTION=="add", SUBSYSTEM=="sound", KERNEL=="controlC[0-9]*", \
  ATTRS{id}=="US16x08", TAG+="systemd", \
  ENV{SYSTEMD_WANTS}+="rackctl-us16x08-default.service"
----

Create the service `/etc/systemd/system/rackctl-us16x08-default.service`. Set
`User=` to your login account: the command then reads *your* default preset from
`~/.config/rackctl/us16x08/default-preset.json`, the same file the tools save. Adjust
`ExecStart` to the path from `command -v` above:

[source,ini]
----
[Unit]
Description=Load the Tascam US-16x08 default mixer preset

[Service]
Type=oneshot
User=YOUR_USERNAME
ExecStart=/usr/local/bin/rackctl-us16x08 default
----

The service is started on demand by the rule, so it is neither enabled nor given
an `[Install]` section. Reload both subsystems to pick up the new files:

[source,bash]
----
sudo udevadm control --reload
sudo systemctl daemon-reload
----

Replug the interface to test. `journalctl -u rackctl-us16x08-default.service` shows whether
the load ran and what it applied. If the open occasionally races the device
becoming ready, add `ExecStartPre=/bin/sleep 1` to the service.

[[sec-gui]]
== Using the graphical mixer

`rackctl-us16x08-gui` opens a single window: a meter bridge across the top, a focused
channel (or linked pair) editor in the centre, an output panel on the right, and
a toolbar to switch between the channel editor, the routing tab, and the preset
tabs (scenes, channel, EQ, and compressor presets). The full list of panels and
controls is in <<sec-gui-ref>>.

[[sec-gui-bridge]]
=== The meter bridge

The meter bridge shows all 16 input channels and the master, always visible at
the top. Each channel column has its name (if set, above the number, truncated
with the full name on hover), a number label, a level meter, and *M* (mute) and
*S* (solo) buttons. Each meter carries a held peak marker (a line that rises with
the level, holds, then falls back slowly; it turns red when the held peak
reached clipping) and a red clip cap at the top that lights briefly when the
meter reaches full scale. Click a channel's number to focus it in the editor below; the
focused channel (or its linked pair) is highlighted. *S* solos the channel: while
any channel is soloed, the others are muted, and clearing solo restores the mutes
that were in place before. The master level meters and the master mute button sit
at the right of the bridge.

[[sec-gui-editor]]
=== The channel editor

The editor shows the focused channel, or one combined strip when the channel's
pair is linked. It has three boxes side by side.

INPUT:: An editable *Name* field for the channel (a GUI-only label such as "Kick"
or "Vox", remembered between sessions and shown in the meter bridge), the
channel's stereo-link checkbox, phase, mute, and solo switches, a vertical volume
fader with its level meter, and a pan slider along the bottom. *Solo* mutes the
other channels (a GUI-only monitoring aid; see the bridge). When the pair is
linked, the pan slider becomes a balance (see <<sec-gui-link>>) and the two
channels share one name and solo together.

EQ:: A `Reset` button and an enable checkbox in the title row, a response curve
showing the combined effect of the four bands, and below it the bands as a Gain /
Freq / Q grid, one row per band (high at the top, low at the bottom). Drag a value
to change it, or click to type. The low and high bands are shelving filters with
no Q, so those cells are blank; the two mid bands are peaking filters with a Q.
`Reset` returns the band controls to their defaults, which gives a flat response. The low and high bands are shelving filters; the two mid bands are
peaking filters. The curve is indicative of the response shape. When the EQ is
disabled (or `DSP bypass` is on) the curve is flat, since the EQ is doing
nothing.

Compressor:: A `Reset` button and an enable checkbox in the title row, a transfer
curve (input level to output level), filled below the line, showing the threshold
and ratio against a 1:1 reference diagonal (make-up gain is a separate output trim
and is not folded into the curve), a vertical gain-reduction meter inside the graph, and the parameters below: threshold, ratio,
and gain on the first row, attack and release on the second. Each is a drag-value
(drag to change, click to type), except ratio, which is a dropdown of the
available ratios. `Reset` returns the parameters to their defaults, a 1:1 ratio
with no make-up gain, so no compression occurs. When the compressor is disabled (or `DSP bypass` is on) the curve is a
straight 1:1 line and the gain-reduction bar reads zero.

All numeric controls read and accept the display units described in
<<sec-controls-model>>; type a value into a control's box to set it directly.

Copy and paste:: There is one clipboard. *Copy channel* (above the boxes) puts the
whole focused strip on it; the EQ and Compressor boxes have their own *Copy* in
their title rows, which overlays just that section on the clipboard. Each *Paste*
applies only the part it handles -- *Paste channel* applies everything on the
clipboard, *Paste* in the EQ box applies just the EQ, and so on. So you can copy a
channel, then copy a different channel's EQ over it, then paste the combination
onto many channels. Only controls that were actually copied are written -- a
section that was never copied is left untouched, and its *Paste* button stays
disabled. Pasting onto a linked pair applies to both channels, and a pasted
channel keeps the pair's hard-left/right panning.

Reset channel:: Next to those, *Reset channel* returns the whole focused channel
to a neutral default -- flat EQ, no compression, centre pan, the fader fully down
(-127 dB) so the channel starts silent, with the phase, mute, and the EQ and
compressor enable switches off. (The EQ and Compressor boxes also have their own
*Reset* buttons for just that section.)

[[sec-gui-output]]
=== The output panel

The output panel on the right carries the master volume fader and the global
switches: `DSP bypass` and `Buss out`, plus the *Save default* and *Load default*
buttons (see <<sec-gui-presets>>). The master mute is in the meter bridge.

[[sec-gui-routing]]
=== The routing tab

The toolbar switches between the *Channel* editor and the *Routing* tab. The
routing tab has one row per physical output (`LINE OUT 1`...`LINE OUT 8`), each
selecting one source: `Master L`, `Master R`, or one of `Output 1`...`Output 8`.

[[sec-gui-scenes]]
=== The scenes tab

The toolbar's *Scenes* tab keeps a list of saved whole-mixer snapshots -- like
the recall scenes of a hardware mixer, but stored as ordinary preset files in the
settings directory, with no fixed limit on how many.

To save the current mixer, type a name and click *Save current mix* (or press
Enter); the snapshot is written to a `<name>.json` file in the scenes directory.
Saving with an existing name overwrites that scene. Each saved scene appears as a
row with three buttons: *Save* overwrites that scene with the current mixer
(updating it in place, keeping its name), *Load* applies it to the whole mixer,
and *Delete* removes the file after a confirmation.

A scene holds the same whole-mixer state as a mixer preset (master, routing, and
all 16 channel strips), plus the stereo-link grouping, so loading it restores
which pairs are linked. It does not carry the interface zoom or window size (those
travel only with the default preset). Scenes are independent of the shared default
preset
(<<sec-gui-presets>>): the default is the one applied automatically on connect,
while scenes are recalled by hand.

[[sec-gui-channel-presets]]
=== The channel presets tab

The *Channel presets* tab is the single-channel counterpart of the scenes tab: it
saves and recalls one channel's settings (fader, pan, phase, mute, EQ, and
compressor) as strip presets stored in the settings directory, with no fixed
limit. It works on the channel currently focused in the editor: *Save this
channel* (or the per-row *Save*) captures that channel, and *Load* applies a
preset to it. As with scenes, each saved preset has *Save* (overwrite), *Load*,
and *Delete*, plus a *Copy* button that copies the preset onto the editor's
clipboard (replacing it, like *Copy channel*) so it can be pasted onto channels
(see <<sec-gui-editor>>).
Because a strip preset is not tied to a channel, you can save from one channel and
load onto another -- the quick way to copy a channel's settings.

[[sec-gui-section-presets]]
=== The EQ and compressor presets tabs

The *EQ presets* and *Comp presets* tabs work just like the channel-presets tab,
but for a single section. *EQ presets* saves and recalls a channel's EQ (the four
bands and the enable switch); *Comp presets* saves and recalls a channel's
compressor (its parameters and enable). Each captures from and applies to the
focused channel, with the same *Save* / *Load* / *Delete* per row. They are handy
for building a library of EQ or compressor settings to drop onto any channel.
Each row also has a *Copy* button that overlays the preset's section onto the
editor's clipboard, ready to *Paste* onto a channel from the editor.

A section preset is stored as a channel strip preset that holds only that
section's controls, so `rackctl-us16x08 load` can apply one to a channel too -- it sets
just the saved controls and leaves the rest of the channel untouched.

Loading a channel, EQ, or compressor preset onto a channel that is part of a
linked pair applies it to both channels of the pair, so the link stays
consistent; the pair's hard-left/right panning is kept.

[[sec-gui-link]]
=== Stereo link

To link a channel pair into one stereo strip, focus either channel and tick the
*Stereo link* checkbox in the INPUT box. The editor then shows one strip, and
each edit is written to both channels. The pair is panned hard L/R, the Volume
fader sets the common level, and the pan control becomes a *Balance* that
attenuates one channel's level to favour the other side (`C` = both at the common
level, full left silences the right channel, full right silences the left).
Unticking the box unlinks the pair. The link state is saved between sessions. See
<<sec-link-concept>> for the concept.

[[sec-gui-presets]]
=== The default preset

Named presets are saved and recalled from their tabs: whole-mixer snapshots from
the <<sec-gui-scenes,Scenes>> tab, and per-channel strips from the
<<sec-gui-channel-presets,Channel presets>> tab.

The output panel's *Save default* and *Load default* buttons capture the current
mixer as the shared default preset, or restore it. This is the same default file
that `rackctl-us16x08 default` uses (see <<sec-cli-default>>). *Save default* also
records the current interface zoom and window size, and *Load default* and the
next startup restore them, so they are part of the saved setup. The zoom and
window size are GUI settings, so `rackctl-us16x08 default` does not change them.

[[sec-gui-keyboard]]
=== Keyboard and zoom

With no control focused, the Left and Right arrow keys move the focused channel,
treating a linked pair as one step; `m` toggles mute on the selected channel, `M`
(Shift+m) toggles the master mute, and `s` toggles solo on the selected channel;
and Esc or Q quits the program. Ctrl with the plus or minus key zooms the whole
interface in or out. The zoom is remembered by *Save default* (see
<<sec-gui-presets>>).

[appendix]
[[sec-appendix-catalog]]
== Control catalog and signal chain (verified reference)

This appendix is the verified description of the device's controls and signal
flow, the single source behind the CLI's `topology` output and the control
catalog. Each claim is tagged for provenance: _(manual)_ from Tascam's published
documentation, _(driver)_ from the Linux `snd-usb-audio` driver
(`sound/usb/mixer_us16x08.c`, authoritative for which controls exist), and
_(inferred)_ reasoned from those but not stated outright. The *Range* column
shows the raw control's `min-max (default)`.

The per-channel stages run in signal order: phase, low-cut filter, EQ,
compressor, mute, fader, pan. The roles are _(manual)_; the EQ-before-compressor
ordering is _(inferred)_ (the manual presents EQ as channel module 1 and the
compressor as module 2, acting on the channel input volume).

=== Phase

[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `phase` | Phase Switch | on / off | Reverse the channel's phase.
|===

=== Low-cut filter

The device has a per-channel low-cut (high-pass) filter, but the Linux driver
registers no control for it, so neither tool can set it. _(driver)_

=== EQ (4 bands)

[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `eq-enable` | EQ Switch | on / off | Enable the channel equalizer.
| `eq-low-volume` | EQ Low Volume | 0-24 (12) | Low band gain; 12 = 0 dB (plus or minus 12 dB).
| `eq-low-freq` | EQ Low Frequency | 0-31 (5) | Low band frequency (32 Hz to 1.6 kHz).
| `eq-midlow-volume` | EQ MidLow Volume | 0-24 (12) | Mid-low band gain (plus or minus 12 dB).
| `eq-midlow-freq` | EQ MidLow Frequency | 0-63 (14) | Mid-low band frequency.
| `eq-midlow-q` | EQ MidLow Q | 0-6 (2) | Mid-low band width (Q 0.25 to 16.0).
| `eq-midhigh-volume` | EQ MidHigh Volume | 0-24 (12) | Mid-high band gain (plus or minus 12 dB).
| `eq-midhigh-freq` | EQ MidHigh Frequency | 0-63 (27) | Mid-high band frequency.
| `eq-midhigh-q` | EQ MidHigh Q | 0-6 (2) | Mid-high band width (Q 0.25 to 16.0).
| `eq-high-volume` | EQ High Volume | 0-24 (12) | High band gain (plus or minus 12 dB).
| `eq-high-freq` | EQ High Frequency | 0-31 (15) | High band frequency (1.6 kHz to 16 kHz).
|===

The band frequency ranges are indicative; only the low band's 32 Hz to 1.6 kHz
span is confirmed by the manual.

=== Compressor

[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `comp-enable` | Compressor Switch | on / off | Enable the channel compressor.
| `comp-threshold` | Compressor Threshold Volume | 0-32 (32) | Threshold; 0 = -32 dB, 32 = 0 dB.
| `comp-ratio` | Compressor Ratio | enum 0-14 (0) | 1.0:1 to inf:1 (15 steps).
| `comp-attack` | Compressor Attack | 0-198 (0) | Attack 2 to 200 ms.
| `comp-release` | Compressor Release | 0-99 (0) | Release 10 to 1000 ms.
| `comp-gain` | Compressor Volume | 0-20 (0) | Make-up gain 0 to 20 dB.
|===

The compressor meters expose input, output, and gain-reduction levels. _(manual)_

=== Mute, fader, pan

[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `mute` | Mute Switch | on / off | Mute the channel.
| `line-volume` | Line Volume | 0-133 (127) | Channel level sent to the stereo bus; logarithmic dB curve.
| `pan` | Pan Left-Right Volume | 0-254 (127) | 0 = hard left, 127 = centre, 254 = hard right; centre is -3 dB to each side.
|===

=== Master bus and routing

The 16 channels are summed into a stereo master; the master meters show the mixed
signal. Each `LINE OUT` selects one source through its own `route` instance.
_(manual)_

[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `master-volume` | Master Volume | 0-133 (127) | Master fader level; logarithmic dB curve.
| `master-mute` | Master Mute Switch | on / off | Mute the master bus.
| `route` | Line Out Route | enum 0-9, per output (0-7) | Source for one physical line output: `Master L`, `Master R`, or `Output 1`...`Output 8`.
|===

The 16 inputs are not routed to outputs individually; they are summed into the
stereo master. Only the eight outputs are routed. `LINE OUT 1` and `LINE OUT 2`
are mirrored to the headphones jack. _(manual)_

=== Global switches and metering

[cols="2,3,2,4",options="header"]
|===
| Control | ALSA name | Range | Notes

| `dsp-bypass` | DSP Bypass Switch | on / off | Bypass the channel DSP (EQ and compressor) for dry monitoring. _(inferred)_
| `buss-out` | Buss Out Switch | on / off | Fold the computer playback into the stereo master bus. _(manual/inferred)_
| `meter` | Level Meter | read-only, 34 ints | Per-channel input levels (0-15), master L/R (16-17), per-channel compressor gain coefficient (18-33; full scale = no reduction, reported as a reduction percentage by the `meters` command).
|===

[appendix]
[[sec-appendix-trouble]]
== Troubleshooting

Card not found:: `rackctl-us16x08` exits with an error and code 1 when no US-16x08 is
present. Confirm the unit is connected and that `aplay -l` (or `cat
/proc/asound/cards`) lists card id `US16x08`. To work without hardware, add
`--mock`.

Meters do not move in the GUI:: Another program may be reading the meters. Close
any running `rackctl-us16x08 monitor` against the same device; run one meter reader at
a time.

The interface was unplugged:: The GUI keeps running. It hides the controls behind
a "Tascam US-16x08 is disconnected" notice while the card is absent and reopens it
automatically once it returns (about a second after the card re-enumerates). On
reconnect it re-applies the mix shown on screen, since the device powers up at its
own defaults. The `rackctl-us16x08` commands are one-shot and instead exit with an error
if the card is gone.

The GUI was started with no interface attached:: It opens to the disconnected
notice and waits. When the card appears it applies the saved default preset (see
<<sec-gui-presets>>), or, if none is saved, simply reads the device's current
settings. When the card is already attached at startup, the GUI reads its current
settings and leaves them unchanged.

A value will not go where expected:: Integer values are in display units and are
clamped to each control's range (see <<sec-appendix-catalog>>). A bare leading `-`
is read as a relative adjust; for a negative absolute value, add a unit suffix
(for example `-6dB`).

A loaded preset reports skipped controls:: Controls the device does not expose
(or that are absent on the running kernel) are skipped on load and listed. The
rest are applied.

[glossary]
== List of used terms

balance:: For a linked stereo pair, the single control that pans the two channels
hard-opposite and tilts that stereo image left or right. Replaces the pan control
in the GUI when a pair is linked.
capture:: The audio the device sends to the computer for recording. Taken before
the DSP block (the dry input).
channel:: One of the 16 inputs, each with its own per-channel DSP chain. Selected
by index 0...15.
control:: One settable value of the mixer, identified by a key, with a scope and a
kind.
default preset:: A mixer preset stored in a fixed location in the user
configuration directory, shared by both tools and recalled with one command or
button.
DSP:: The on-board digital signal processing (EQ, low-cut filter, compressor) that
shapes monitoring and the outputs, not the recording.
gain reduction:: The amount, in dB, by which the compressor is currently lowering a
channel's level. Shown as a meter.
key:: The short name of a control used by `rackctl-us16x08` (for example
`master-volume`).
LCF:: Low-cut filter; a per-channel high-pass filter present on the device but not
exposed by the Linux driver.
master bus:: The stereo mix into which the 16 channels are summed.
meter:: A read-only level reading. The meter block carries per-channel levels, the
master level, and per-channel gain reduction.
mixer preset:: A saved file holding the whole mixer: master, routing, and all 16
channel strips.
mock device:: An in-memory stand-in for the hardware, selected with `--mock`, so
the tools run with no card attached.
route:: The per-output control that selects which source a physical line output
carries.
scene:: A named whole-mixer snapshot saved in the settings directory and recalled
by hand from the GUI's scenes tab. The host-side equivalent of a hardware mixer's
scene memory.
scope:: Whether a control is global, per channel, or per output.
strip preset:: A saved file holding one channel's controls, applicable to any
channel.
stereo link:: A GUI grouping of an adjacent channel pair into one stereo strip;
not a hardware control.
