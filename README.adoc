= Rackctl — control your rack from Linux
:toc:
:icons: font
:source-highlighter: rouge
:experimental:

Rackctl is a Rust workspace of Linux utilities for controlling outboard audio
hardware. Its first member is a pair of tools for the *Tascam US-16x08* USB
interface's on-board DSP mixer (faders, EQ, compressor, routing, metering),
which the `snd-usb-audio` driver exposes as ~280 ALSA control elements; the tools
drive that surface — there is no PCM streaming involved. Support for further
devices (a BOSS GX-700 is planned) will be added as sibling crates.

The US-16x08 tools comprise:

. *Library* — `crates/us16x08` (`rackctl-us16x08`): a pure, UI-agnostic crate
  wrapping the ALSA control surface. *(done)*
. *CLI* — `crates/us16x08-cli`: the `rackctl-us16x08` command-line tool. *(done)*
. *GUI* — `crates/us16x08-gui`: the `rackctl-us16x08-gui` graphical mixer
  (egui/eframe). *(done)*

== Using the tools

For installing and operating both tools, see
link:docs/user-manual.adoc[the user manual]. It covers the signal flow, the
command-line tool `rackctl-us16x08`, the graphical mixer `rackctl-us16x08-gui`, presets, and
the verified control catalog.

A global `--mock` flag (CLI) or `--mock` argument (GUI) runs against an in-memory
device with no card attached, e.g. `cargo run -p rackctl-us16x08-cli -- --mock list`.

=== With Nix (flake)

The repository is a Nix flake, so on a system with Nix and flakes enabled you can
build or run the tools without installing a Rust toolchain or the eframe system
libraries by hand — the flake pulls in `alsa-lib`, `libGL`, `libxkbcommon`,
Wayland, and the X11/xcb libraries, and wraps `rackctl-us16x08-gui` so it finds the
libraries it loads at run time.

[source,bash]
----
nix run github:malo-rte/rackctl                       # run the GUI
nix run github:malo-rte/rackctl#rackctl-us16x08 -- list   # run the CLI
nix build github:malo-rte/rackctl                     # build into ./result
nix develop                                           # dev shell: cargo, clippy, the gate
----

The package version is read from the workspace `Cargo.toml`, so it tracks
releases automatically. Run `nix flake lock` once to generate and commit
`flake.lock`, pinning `nixpkgs`.

=== Desktop integration and packaging

A desktop entry for the GUI and shell completions for `rackctl-us16x08` (generated by
`rackctl-us16x08 completions <shell>`) are in `packaging/`; see
link:packaging/README.adoc[packaging/README.adoc] for a manual install. The Nix
package installs both automatically.

[source]
----
Cargo.toml                 # Rust workspace (+ shared lint policy)

crates/
  us16x08/                 # the control-surface library (rackctl-us16x08)
  us16x08-cli/             # the `rackctl-us16x08` command-line tool
  us16x08-gui/             # the `rackctl-us16x08-gui` graphical mixer
packaging/                 # desktop entry + packaging notes
----

== The library (`rackctl-us16x08`)

A typed wrapper over the ALSA high-level control (HCTL) interface, the Rust port
of the C++ `OAlsa` hardware layer.

* `Control` enumerates every DSP/mixer control with its ALSA name, value kind,
  range, and scope (global / per-channel / per-output).
* `Us16x08<B>` is the device façade, generic over a `Backend`:
** `AlsaBackend` (feature `alsa`, default) talks to real hardware.
** `MockBackend` is an in-memory stand-in needing no card or `libasound`, so the
   full surface can be exercised in tests and on CI.
* `Meters` decodes the 34-value level-meter block; `convert` ports the fader/meter
  dB curves; `Watcher` reports external control changes.

=== Building and testing

The default build links `libasound` via the `alsa` crate (install `libasound2-dev`
on Debian/Ubuntu, `alsa-lib` on Arch):

[source,bash]
----
cargo build
cargo test
----

To build and test without a sound card or ALSA headers (mock backend only):

[source,bash]
----
cargo test -p rackctl-us16x08 --no-default-features
----

=== The full gate

`./run-all-checks.sh` runs the same checks as CI: `cargo fmt --check`,
`clippy -D warnings` and `cargo test` over both feature sets, `rustdoc -D warnings`,
and — when installed — `cargo deny`, `shellcheck`, and `shfmt`. It runs every check
and reports all failures, exiting non-zero if any fails. CI
(`.github/workflows/ci.yml`) additionally builds and tests on the MSRV (1.95).

== Documentation

Docs live in `docs/` as AsciiDoc. Render them to PDF (under `build/docs/`) with:

[source,bash]
----
./docs/build.sh
----

This needs `asciidoctor-pdf` and `asciidoctor-diagram` (Ruby gems) plus `plantuml`
(and `graphviz`) for the embedded figures:

[source,bash]
----
sudo gem install asciidoctor-pdf asciidoctor-diagram
sudo apt-get install plantuml graphviz
----

== Acknowledgments

This project was inspired by
https://github.com/onkelDead/tascam-gtk[tascam-gtk] by onkelDead, a GTK+
application for the same interface. That project showed the US-16x08's DSP mixer
could be driven well from Linux and informed the feature set here. This is an
independent Rust rewrite — a fresh implementation rather than a port or a copy of
its code — built around a focused one-channel-at-a-time editor and a typed control
library. With thanks to its author.
