This repository is a Rust rewrite of the Tascam mixer. The device's DSP mixer (faders, EQ,
compressor, routing, metering) is exposed by the Linux `snd-usb-audio` driver as
~280 ALSA control elements; the project drives that surface â€” there is no PCM
streaming involved.

The work proceeds in three stages:

. *Library* â€” `crates/tascam-us16x08`: a pure, UI-agnostic crate wrapping the ALSA
  control surface. *(done)*
. *CLI* â€” `crates/tascam-cli`: the `tascamctl` command-line tool. *(done)*
. *GUI* â€” `crates/tascam-gui`: the `tascam-mixer` graphical mixer (egui/eframe).
  *(done)*

== Using the tools

For installing and operating both tools, see
link:docs/user-manual.adoc[the user manual]. It covers the signal flow, the
command-line tool `tascamctl`, the graphical mixer `tascam-mixer`, presets, and
the verified control catalog.

A global `--mock` flag (CLI) or `--mock` argument (GUI) runs against an in-memory
device with no card attached, e.g. `cargo run -p tascam-cli -- --mock list`.

=== With Nix (flake)

The repository is a Nix flake, so on a system with Nix and flakes enabled you can
build or run the tools without installing a Rust toolchain or the eframe system
libraries by hand â€” the flake pulls in `alsa-lib`, `libGL`, `libxkbcommon`,
Wayland, and the X11/xcb libraries, and wraps `tascam-mixer` so it finds the
libraries it loads at run time.

[source,bash]
----
nix run github:malo-rte/tascam-mixer                 # run the GUI
nix run github:malo-rte/tascam-mixer#tascamctl -- list   # run the CLI
nix build github:malo-rte/tascam-mixer               # build into ./result
nix develop                                          # dev shell: cargo, clippy, the gate
----

The package version is read from the workspace `Cargo.toml`, so it tracks
releases automatically. Run `nix flake lock` once to generate and commit
`flake.lock`, pinning `nixpkgs`.

=== Desktop integration and packaging

A desktop entry for the GUI and shell completions for `tascamctl` (generated by
`tascamctl completions <shell>`) are in `packaging/`; see
link:packaging/README.adoc[packaging/README.adoc] for a manual install. The Nix

== The library (`tascam-us16x08`)

A typed wrapper over the ALSA high-level control (HCTL) interface, the Rust port
of the C++ `OAlsa` hardware layer.

* `Control` enumerates every DSP/mixer control with its ALSA name, value kind,
  range, and scope (global / per-channel / per-output).
* `Us16x08<B>` is the device faĂ§ade, generic over a `Backend`:
** `AlsaBackend` (feature `alsa`, default) talks to real hardware.
** `MockBackend` is an in-memory stand-in needing no card or `libasound`, so the
   full surface can be exercised in tests and on CI.
* `Meters` decodes the 34-value level-meter block; `convert` ports the fader/meter
  dB curves; `Watcher` reports external control changes.

=== Building and testing

The default build links `libasound` via the `alsa` crate (install `libasound2-dev`
on Debian/Ubuntu, `alsa-lib` on Arch):

[source,bash]
----
cargo build
cargo test
----

To build and test without a sound card or ALSA headers (mock backend only):

[source,bash]
----
cargo test -p tascam-us16x08 --no-default-features
----

=== The full gate

`./run-all-checks.sh` runs the same checks as CI: `cargo fmt --check`,
`clippy -D warnings` and `cargo test` over both feature sets, `rustdoc -D warnings`,
and â€” when installed â€” `cargo deny`, `shellcheck`, and `shfmt`. It runs every check
and reports all failures, exiting non-zero if any fails. CI
(`.github/workflows/ci.yml`) additionally builds and tests on the MSRV (1.85).

== Documentation

Docs live in `docs/` as AsciiDoc. Render them to PDF (under `build/docs/`) with:

[source,bash]
----
./docs/build.sh
----

This needs `asciidoctor-pdf` and `asciidoctor-diagram` (Ruby gems) plus `plantuml`
(and `graphviz`) for the embedded figures:

[source,bash]
----
sudo gem install asciidoctor-pdf asciidoctor-diagram
sudo apt-get install plantuml graphviz
----
