= Packaging and desktop integration

Distribution assets for the Rackctl suite and notes on installing them by hand.
The suite ships three devices, each with a CLI and a GUI:

[cols="1,1,2",options="header"]
|===
| Device | Binaries | What it controls
| Tascam US-16x08 | `rackctl-us16x08`, `rackctl-us16x08-gui` | The USB interface's built-in DSP mixer (ALSA)
| BOSS GX-700 | `rackctl-gx700`, `rackctl-gx700-gui` | Guitar-effects patches over MIDI SysEx
| Avid/Digidesign Eleven Rack | `rackctl-eleven`, `rackctl-eleven-gui` | Guitar-amp patches over MIDI SysEx
|===

The Nix flake (`../flake.nix`) wires all of this up automatically — `nix profile
install github:malo-rte/rackctl` gives you every binary, both desktop entries, and
all shell completions. The steps below are for packagers and for a manual install.

== Binaries

Build the release binaries and copy them onto the `PATH`:

[source,bash]
----
cargo build --release --workspace
for bin in rackctl-us16x08 rackctl-us16x08-gui rackctl-gx700 rackctl-gx700-gui \
           rackctl-eleven rackctl-eleven-gui; do
  sudo install -Dm755 "target/release/$bin" "/usr/bin/$bin"
done
----

== Desktop entries

Each GUI has a `.desktop` entry that launches it from a desktop menu or launcher.
Its `StartupWMClass` matches the GUI's Wayland `app_id` (the binary name), so
compositors associate the window with the entry.

[source,bash]
----
sudo install -Dm644 -t /usr/share/applications packaging/*.desktop
----

Each entry references an icon named after its binary (`rackctl-us16x08-gui`,
`rackctl-gx700-gui`). The matching scalable (SVG) icons ship alongside the
desktop entries; install them into the hicolor icon theme so launchers show
them:

[source,bash]
----
for gui in rackctl-us16x08-gui rackctl-gx700-gui rackctl-eleven-gui; do
  sudo install -Dm644 "packaging/$gui.svg" \
    "/usr/share/icons/hicolor/scalable/apps/$gui.svg"
done
----

== Shell completions

Each CLI generates its own completion scripts, so they always match the current
commands and flags. Generate and install them for the shells you want — the
example does both CLIs:

[source,bash]
----
for cli in rackctl-us16x08 rackctl-gx700 rackctl-eleven; do
  # bash
  "$cli" completions bash | sudo tee "/usr/share/bash-completion/completions/$cli" >/dev/null
  # zsh  (a directory on your $fpath)
  "$cli" completions zsh  | sudo tee "/usr/share/zsh/site-functions/_$cli" >/dev/null
  # fish
  "$cli" completions fish > "$HOME/.config/fish/completions/$cli.fish"
done
----

Supported shells: `bash`, `zsh`, `fish`, `elvish`, `powershell`.
